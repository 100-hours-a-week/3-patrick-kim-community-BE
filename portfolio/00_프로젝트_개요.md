# PetStar - 반려동물 사진 콘테스트 플랫폼

## 프로젝트 소개

**PetStar**는 반려동물 사진 콘테스트 플랫폼입니다. 사용자들이 자신의 반려동물 사진을 챌린지에 출품하고, 다른 사용자들이 투표하여 실시간 랭킹을 확인할 수 있습니다.

### 핵심 기능
- 챌린지(콘테스트) 생성 및 관리
- 반려동물 사진 출품 (Entry)
- 실시간 투표 및 랭킹
- 응원 메시지

### 기술 스택
- **Backend**: Spring Boot 3.5, Java 21
- **Database**: MySQL 8.0 (AWS RDS)
- **Cache**: Redis 7 (Sorted Set)
- **Infra**: AWS EC2, Terraform, Docker
- **Monitoring**: Prometheus, Grafana, p6spy

---

## 왜 이 프로젝트인가?

### 투표 기반 랭킹 시스템의 기술적 도전

PetStar는 단순한 CRUD 애플리케이션이 아닙니다. **투표 결과가 실시간으로 랭킹에 반영**되어야 하고, **그 랭킹이 콘테스트 우승자를 결정**합니다.

```
PetStar의 기술적 특성:

1. 트래픽 집중
   - 챌린지 종료 직전 투표 급증
   - 인기 Entry에 투표 집중 (Hot Spot)

2. 데이터 정합성 요구
   - 1표 = 순위 변동 가능
   - 투표 누락 또는 중복 불가

3. 실시간성 요구
   - 투표 후 즉시 랭킹 반영
   - 사용자가 결과를 바로 확인
```

이 조합이 만들어내는 문제들:
- **동시 투표 시 Race Condition** → Lost Update, 중복 투표
- **매 요청 DB 정렬** → 트래픽 증가 시 DB 병목
- **Hot Spot에서의 동시성** → 일반적인 해결책이 동작하지 않음

### 목표: "동작하는 코드"를 넘어서

| 목표 | 기준 |
|------|------|
| 성능 | p95 응답시간 < 200ms |
| 확장성 | 동시 접속자 200명 안정 처리 |
| 정합성 | 투표 누락/중복 0% |

### 접근 방식: 측정 → 분석 → 해결 → 검증

```
1. 측정: "느리다"가 아니라 "어디서 얼마나 느린가"
   - k6 부하테스트로 수치화
   - Grafana 메트릭으로 병목 시각화
   - p6spy 쿼리 로그로 원인 추적

2. 분석: "왜 이 문제가 발생하는가"
   - JPA Lazy Loading 메커니즘 이해
   - 트랜잭션 격리 수준과 락 동작 이해
   - 인덱스 내부 구조와 실행 계획 분석

3. 해결: "여러 대안 중 최선은 무엇인가"
   - 트레이드오프 비교
   - PetStar 서비스 특성에 맞는 선택
   - 구현 및 테스트

4. 검증: "실제로 개선되었는가"
   - 동일 조건에서 재측정
   - 수치로 증명
```
